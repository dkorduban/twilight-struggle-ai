# Opus Analysis: results/ Directory Restructuring
Date: 2026-04-11 ~21:30 UTC
Question: Restructure results/ into topic subdirs, fix hardcoded paths

## Executive Summary

The results/ directory contains 274 files in a flat dump (189 logs, 55 MDs, 8 JSONs, 21 TXTs, 1 PNG) plus 3 existing subdirs (archive/, bench_jobs/, matches/). Snakefile already uses results/logs/ and results/bench/ subdirs. The proposed restructuring moves files into 7 topic subdirs (logs/ppo/, logs/elo/, logs/training/, logs/misc/, analysis/, elo/, bench/) using git mv. Only 4 scripts contain hardcoded paths that need updating after moves: ppo_loop_step.sh (elo log path), run_elo_tournament.py (default --out), compute_elo.py (default --out), and select_league_fixtures.py (elo/jsd paths). The bench/ and logs/ subdirs already exist and are used by Snakefile, so those moves are additive, not breaking.

## Conclusions

1. **274 files in flat results/ can be organized into 7 topic subdirs** with zero breaking changes to active infrastructure (Snakefile already uses the target subdirs).
2. **~250 files move** (65 PPO logs, 70 Elo logs, 32 training logs, 10 misc logs, 7 Elo JSONs, 54 analysis MDs, 22 archived TXTs), leaving only 5 frequently-written files in the root.
3. **12 scripts need path updates** across ~20 specific lines. The highest-impact script is ppo_loop_step.sh (6 line changes), which is the active PPO pipeline driver.
4. **The Snakefile needs zero changes** -- it already writes to results/logs/ and results/bench/.
5. **No Python library code** (python/tsrl/) references results/ paths, so no library changes needed.
6. **All moves use git mv** to preserve history.

2. **benchmark_history.json:** Referenced by 8+ scripts. Moving to results/elo/ would require many edits. Recommend leaving in root. Alternatively, could add a symlink.

No conflict, but the Snakefile logs will remain at the results/logs/ level alongside the new subdirs.

4. **Should historical/one-off scripts (run_v110_data_sweep.sh, run_v111_experiments.sh, etc.) be updated?** Their output files are being archived and they are unlikely to be run again. Recommend: no changes to historical scripts.
