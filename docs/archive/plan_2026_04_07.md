## What We Learned Today

1. **PPO v3 vs heuristic ceiling is ~90%**. Heuristic WR gains have diminished returns — further progress requires better opponents or fundamentally better play.

2. **Heuristic WR is a poor proxy above 80%**. v2b iter140 (83.6%) was rated 1823 BayesElo while v1_best (83.2%) was 1735 — 88 Elo apart despite similar WRs. H2H matchups are essential.

3. **n_steps drop in v3 is structural, not a bug**: league records only the learning model's side (~12k steps/iter vs ~30k in self-play). Not declining over training.

4. **The GNN vs Attention comparison from Phase 1 was confounded**: GNN had region scalars + side embedding + stability features that Attention lacked. A fair comparison hasn't been done yet.

5. **Active effects (21 boolean features) are ready to activate**: bear_trap, quagmire, CMC, flower_power, NATO, etc. These are strategically important features the model currently cannot see.

### Phase 3: Collect v3-quality self-play dataset (Day 1, ~2 hours)

**Goal**: Build a BC training dataset from strong models, not heuristic play.

**Why not nash_bcd_combined (4.08M heuristic rows)?**
- Heuristic plays a narrow subset of strategies: barely uses space race, rarely triggers
  bear traps, CMC, quagmire, or other active effects
- The new 21 scalar features will be almost all zeros in heuristic data
- Architecture comparison on heuristic data tests "which model learns heuristic patterns"
  not "which model learns good TS play"
- v3-level self-play produces strategically rich data where active effects actually fire

**Data sources** (all use 32-dim scalars from Phase 1):

| Source | Games | Est. rows | Collection time | Quality |
|--------|-------|-----------|----------------|---------|
| v3_best vs v3_best | 4000 | ~520k | ~6 min | Highest: balanced, diverse |
| v3_best vs league pool | 4000 | ~520k | ~6 min | High: varied opponents |
| v3_best vs heuristic | 4000 | ~240k* | ~6 min | Medium: one-sided recording |
| v2b checkpoints vs each other | 8000 | ~1.0M | ~11 min | Medium: past strategies |

*One-sided recording for model-vs-heuristic.

**Collection plan**: Build the dataset in tiers so we can test data scaling:

| Tier | Content | Total rows | Collection time |
|------|---------|-----------|----------------|
| 0.5M | v3 self-play (4k games) | ~520k | ~6 min |
| 1M | + v3 vs league pool (4k games) | ~1.04M | ~12 min |
| 2M | + v2b cross-play (8k games) | ~2.04M | ~23 min |
| 4M | + v3 vs heuristic (8k) + more v3 SP (8k) | ~4.1M | ~44 min |
| 8M | + v2b vs heuristic (16k) + more cross-play (16k) | ~8.3M | ~89 min |

Collection order matters: each tier adds progressively lower-quality data.
Tier 0.5M is the purest (v3 self-play only). Tier 8M mixes in weaker play for volume.

**Implementation**: Use `rollout_self_play_batched` (records both sides) and
`rollout_model_vs_model_batched` (records model_a only) with 32-dim scalars.
Write a `scripts/collect_bc_dataset.py` that:
- Takes a list of checkpoint paths + game counts
- Collects in batches of 200 games
- Writes per-batch parquet files to `data/v3_selfplay/`
- Records metadata (source checkpoints, seeds, row counts)

### Phase 4: Architecture comparison with data scaling (Days 2-3)

**Goal**: Determine the best architecture AND how much data it needs.

**Design**: Full factorial — 2 architectures × 5 dataset sizes × 3 seeds = 30 runs.
But we can early-stop: if one architecture dominates at 0.5M and 1M, skip larger sizes.

**Architectures**:
1. **GNN** (`control_feat_gnn_side`): 2-hop message passing over adjacency graph,
   region scalars, side embedding. Current production architecture.
2. **Attention** (`country_attn_side`): NEW — clone `TSCountryAttnModel`, add
   region scalars + side embedding + stability features to match GNN's feature set.
   This isolates the adjacency-vs-attention question.

**Protocol**:

```
Round 1: Train at 0.5M (v3 self-play only)
  - GNN seeds {7, 42, 123}
  - Attn seeds {7, 42, 123}
  - ~15 min/run × 6 = ~90 min GPU
  - Compare val_loss, card_top1, mode_acc

Round 2: Train at 1M (+ v3 vs league)
  - Same 6 configs
  - ~30 min GPU total
  - If gap is widening: GNN wins, stop here
  - If gap is narrowing: Attention benefits from data, continue

Round 3 (conditional): Train at 2M
  - Only if Round 2 gap < 3pp
  - ~60 min GPU total

Round 4 (conditional): Train at 4M
  - Only if Round 3 gap < 2pp (attention still closing)
  - ~120 min GPU total

Round 5 (conditional): Train at 8M
  - Only if Round 4 gap < 1.5pp (rare — would mean attention is competitive)
  - ~240 min GPU total
```

**Decision criteria** (per round):
- If GNN mean val_loss < Attn mean val_loss by > 1 seed-std: GNN wins at this size
- If gap is growing across sizes: GNN definitively better, stop
- If gap is shrinking across sizes: attention benefits from scale, continue to next tier
- At any tier, if Attention beats GNN: consider GNN+Attention hybrid

**Hyperparams** (same for all runs):
```
bs=8192, lr=0.0024, epochs=60, patience=15, h=256,
dropout=0.1, weight_decay=1e-4, label_smoothing=0.05,
one_cycle=True, value_target=final_vp
```

**Expected outcome**: GNN wins at all sizes. The adjacency graph encodes legal
influence placement rules — self-attention must learn spatial structure from data,
while GNN gets it for free. But if attention narrows the gap with more data, a
GNN+Attention hybrid becomes interesting.

### Phase 5: Proxy eval set (Day 2, parallel with Phase 4)

**Goal**: Fast feedback loop for architecture and hyperparameter experiments.

**Steps**:
1. Build proxy eval set: 5000 states from the v3 self-play data (0.5M tier),
   stratified by turn/side, max 5/game
2. Write `scripts/eval_proxy.py` to evaluate any checkpoint on proxy set in <3s
3. Calibrate proxy vs 500-game benchmark using Phase 4's checkpoints
4. Use proxy as Tier 1 metric for all future experiments

### Phase 6: PPO v4 with best architecture + 32-dim scalars (Days 4-7)

**Goal**: Push past v3's ~1850 BayesElo.

**Steps**:
1. Train BC on the largest useful data tier from Phase 4 (likely 1-2M of v3-quality data)
2. Export as TorchScript
3. Start PPO v4:
   ```
   --league data/checkpoints/league_v4 --league-save-every 20
   --vp-reward-coef 0.1 --ent-coef-final 0.001 --lr 3e-5
   --games-per-iter 400  (doubled, compensating for 1-sided recording)
   --save-rollout-parquet data/ppo_v4_rollouts/
   ```
4. Seed league pool with v3's league checkpoints (copy iter_0001..iter_0200 to league_v4/)
5. Benchmark every 20 iters + H2H vs v3_best every 40 iters
6. Run for 200 iters (~3 days at ~35s/iter with 400 games)

**Key changes from v3**:
- 32-dim scalars (21 new active-effect features)
- BC base trained on v3-quality self-play data (not heuristic data)
- `--games-per-iter 400` to compensate for one-sided recording (~24k steps/iter)
- League pool seeded with v3 checkpoints (diverse initial opponents)
- end_reason fix: Wargames correctly labeled, VP shaping for Europe control/Wargames
